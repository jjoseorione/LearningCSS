Herencia de propiedades de estilo

Descripción

CSS significa Hojas de Estilo en Cascada, lo que quiere decir justamente que la forma en que se definen los estilos es en forma de cascada. Esto significa que podemos ir de lo general a lo particular y posteriormente a lo específico. por ejemplo el selector más general en CSS es llamado Selector global y se define con un asterísco (*). Posteriormente tenemos el selector del elemento body, y depués estarían los elementos h1, h2, h3, p, div, etc., según cómo se encuentren anidados.

La utilidad de las hojas en cascada radica en la herencia que se logra. Por ejemplo, podríamos definir una regla CSS que indique que todo el elemento body debe llevar fuente monospace. Al hacer esto, ése será el estándar para todo el elemento body y la única forma de elegir otro tipo de fuente (por ejemplo, serif) será hacerlo específicamente en el elemento deseado. De la misma manera, cualquier elemento HTML hereda las reglas de todos sus ancestros y la única forma de cambiarlos es hacerlo de forma específica en dicho elemento.

Ejemplo

Ejemplo de herencia en CSS. Como puede notarse, se le dan algunas reglas de estilo al elemento *, lo que hará que todos los elementos hijos sigan dichas reglas. Sin embargo, a algunos títulos se les ha permitido tener alguna regla específica:

<!DOCTYPE html>
<html>
<head>

<title></title>
<meta charset="UTF-8">
<style>
*
{
color: blue;
font-family: monospace;
font-size: 16px;
font-weight: normal;
}
h2
{
font-size: 20px;
}
h3
{
color: tomato;
}
h4
{
font-family: sans-serif;
}
</style>

</head>
<body>

<h1>Título de nivel 1 con todas las reglas de *</h1>
<h2>Título de nivel 2 con su propia regla de tamaño</h2>
<h3>Título de nivel 3 con su propia regla de color</h3>
<h4>Título de nivel 4 con su propia regla de fuente</h4>
<p>
Párrafo con todas las reglas de *
</p>

</body>
</html>
